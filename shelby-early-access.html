Architecture

How Shelby
serves data

⚡
Hot Write Path

Builders push data — AI outputs, logs, posts, media — directly to Shelby's write API. Data is immediately available for streaming, no propagation delay.

🌐
Geo-distributed Nodes

Shelby replicates data across geo-distributed storage nodes. Reads are served from the closest node, delivering CDN-like performance without centralization.

🔄
Live Stream API

Frontend apps subscribe to data streams via Shelby's stream API. New writes propagate instantly to active subscribers — no polling, no websocket servers to manage.

🔗
Aptos-native

Shelby is designed around the Aptos ecosystem. On-chain events and Move contract state can be mirrored into Shelby for instant off-chain access.
